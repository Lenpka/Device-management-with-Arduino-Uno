# Многоканальная система управления исполнительными устройствами (Arduino UNO + Python)

## 1. Название и цель (научный стиль)

**Название:** «Разработка программно-аппаратного комплекса дистанционного многоканального управления исполнительными устройствами на базе микроконтроллера Arduino UNO и транзисторного драйвера с верхним уровнем на Python».

**Цель:** спроектировать и реализовать двухуровневую систему, в которой прикладной уровень (Python) формирует команды управления по интерфейсу UART (USB Serial), микроконтроллер (Arduino UNO) преобразует их в цифровые сигналы на выводах GPIO, а 8-канальный транзисторный драйвер (ULN2803A или функциональный аналог) коммутирует электрические цепи нагрузок; обеспечить режимы статического включения/выключения, частотного переключения, ШИМ (на выводах с аппаратным PWM) и телеметрию состояния каналов.

**Задачи (для отчёта/защиты):**
- описать архитектуру и обосновать выбор драйвера нагрузки;
- разработать протокол обмена Python ↔ Arduino;
- реализовать прошивку и клиентское ПО;
- провести испытания на согласованных нагрузках (светодиодные индикаторы, малые DC-двигатели 5–12 В, электромагниты и т.п. в пределах паспорта драйвера).

## 2. Архитектура системы

```
[Оператор / ПК]
       │
       ▼
[Python: GUI или скрипт]  —— USB Virtual COM ——►  [Arduino UNO: парсер команд, логика режимов]
                                                       │
                                                       │ 8× логический сигнал 0/3.3–5 В (TTL)
                                                       ▼
                                            [ULN2803A: 8 каналов, ключи «на землю»]
                                                       │
                                                       ▼
                                            [Нагрузки: лампы/LED, соленоиды, моторы DC и т.д.]
                                                       │
                                            питание нагрузок: отдельный источник V_load (+)
                                            общий GND: Arduino ↔ драйвер ↔ источник V_load
```

**Почему схема технически верна:**
- Arduino не рассчитан на прямое питание мощных нагрузок с выводов GPIO (ограничение тока ~20–40 mA на пин). Драйвер ULN2803A принимает малый управляющий ток и коммутирует до сотен mA на канал (по даташиту, с учётом суммарного тока и нагрева).
- Разделение «логика 5 В» и «силовая часть V_load» снижает риск помех и перегрузки МК.
- Python на ПК удобен для интерфейса, логирования и сценариев; жёсткие тайминги переключения выполняет Arduino.

## 3. Роль блоков

| Блок | Функция |
|------|---------|
| **Python** | Формирование команд, отображение состояния, сценарии (последовательности, частоты), запрос телеметрии. |
| **USB Serial (UART)** | Физический и логический канал 115200 бод (типично для UNO). Текстовые команды — простота отладки в Serial Monitor. |
| **Arduino UNO** | Парсинг строк, конечный автомат по каналам (OFF / ON / FREQ / PWM), неблокирующие таймеры (`millis()`), ответы `ACK` / `STAT`. |
| **ULN2803A** | 8 транзисторных ключей с общим эмиттером (open-collector): при HIGH на входе канала ток нагрузки течёт через вывод драйвера в GND. |
| **Нагрузка** | Потребитель, подключённый между `+V_load` и выходом канала драйвера. Конкретный тип выбирается под учебный стенд (см. раздел «Уточнения»). |

**Почему не «абстрактное реле»:** реле — отдельный класс устройств (гальваническая развязка, щелчок, ресурс контактов). В проекте базовый элемент — **канал транзисторного драйвера**; реле можно подключить *как нагрузку* канала, если это обосновано на защите, но архитектура описывается через драйвер.

## 4. Список компонентов

### Обязательные
- Arduino UNO R3 (или совместимая плата ATmega328P + USB-UART).
- Модуль или микросхема **ULN2803A** (готовая breakout-плата предпочтительна для пайки/монтажа).
- USB-кабель A–B (данные, не только зарядка).
- Источник питания нагрузок **V_load** (5 V или 12 V — согласовать с нагрузками; **не** питать моторы только от 5 V пина Arduino).
- Общая шина GND: Arduino, драйвер, минус V_load.
- 8 проводников «вход драйвера» от GPIO Arduino к IN1…IN8.
- Нагрузки для демонстрации: 8× LED + резисторы **или** 8 одинаковых малых исполнительных элементов в пределах тока канала.
- ПК с Python 3.10+.

### Желательные
- Макетная плата, перемычки.
- Предохранитель или полифуз на линии V_load.
- Диоды 1N4007 (flyback) **параллельно катушкам** индуктивных нагрузок (соленоиды, реле) — катод к +V_load.
- Логический анализатор / второй USB-UART для контроля линии (не обязательно на защите).

### Опциональные
- GUI (Tkinter / PyQt) поверх того же протокола.
- Дисплей на стенде, кнопки локального ручного режима.
- Датчик тока на один канал (демонстрация диагностики).
- Внешнее питание 7–12 V для Arduino через разъём barrel (если USB не хватает при многих LED).

## 5. Протокол обмена (кратко)

Формат: **текстовые строки ASCII**, окончание `\n`, скорость **115200 8N1**.

| Команда (ПК → Arduino) | Пример | Действие |
|------------------------|--------|----------|
| `SET <ch> <0\|1>` | `SET 3 1` | Канал `ch` в статический ON/OFF |
| `FREQ <ch> <Hz>` | `FREQ 2 5` | Квадратный сигнал на канале; `Hz=0` — остановить |
| `PWM <ch> <0..255>` | `PWM 4 128` | ШИМ на канале (только если пин поддерживает hardware PWM) |
| `GET` | `GET` | Запрос телеметрии |

| Ответ (Arduino → ПК) | Пример | Смысл |
|----------------------|--------|--------|
| `ACK ...` | `ACK SET 3 1` | Команда принята |
| `ERR ...` | `ERR PWM 1` | Ошибка (неверный канал, PWM недоступен) |
| `STAT ...` | `STAT 1,0,1,0,0,0,0,0` | Логическое состояние выходов 1…8 (1 = ключ открыт / «включено») |
| `MODE ...` | `MODE 2,FREQ,5` | Режим канала (опционально в расширении) |

Полная спецификация: [PROTOCOL.md](PROTOCOL.md).

## 6. Что такое «канал 1, 2, … 8»

**Канал N** — это одна цепочка управления:

1. **Один цифровой вывод Arduino** (см. таблицу в прошивке).
2. **Один вход INn микросхемы ULN2803A** (IN1 → канал 1, …, IN8 → канал 8).
3. **Один выход OUTn драйвера**, к которому подключена **одна независимая нагрузка** (или группа нагрузок, считающаяся одним исполнительным устройством).

Нумерация каналов в протоколе: **1…8** (для человека); в коде Arduino индекс **0…7**.

| Канал | Пин Arduino | Hardware PWM (ATmega328) |
|-------|-------------|---------------------------|
| 1 | D2 | нет |
| 2 | D3 | да |
| 3 | D4 | нет |
| 4 | D5 | да |
| 5 | D6 | да |
| 6 | D7 | нет |
| 7 | D8 | нет |
| 8 | D9 | да |

**Подключение нагрузки (типовое для ULN2803):** `+V_load` → нагрузка → `OUTn` → внутри драйвера → GND. Вход `INn` ← пин Arduino (через резистор обычно не требуется для ULN2803).

## 7. Частотное переключение и телеметрия

### Частотное переключение
- Реализуется **программно на Arduino**: период `T = 1000 / (2 × f)` мс, переключение вывода по `millis()` (неблокирующий цикл).
- Подходит для мигания индикаторов, пульсации соленоида, формирования «мигания тревоги».
- Верхняя частота ограничена разрешением `millis()` (~десятки–сотни Гц на практике для 8 каналов одновременно; для защиты указать измеренный предел на стенде).

### PWM
- На UNO **аппаратный PWM** (`analogWrite`) только на пинах **3, 5, 6, 9** в данной разводке — каналы **2, 4, 5, 8**.
- Для каналов без hardware PWM команда `PWM` возвращает `ERR`.
- ULN2803 работает как ключ: «PWM» даёт **изменение средней мощности** за счёт скважности (для ламп/LED/нагрева), не полноценный сервоприводный сигнал.

### Телеметрия
- Python периодически шлёт `GET`.
- Arduino отвечает строкой `STAT` с восемью значениями 0/1 — **логическое командное состояние** канала (включая режим FREQ в момент опроса — «текущий уровень на выводе» или «целевое ON» — в прошивке примера фиксируется **фактический уровень на пине**).
- Для отчёта: логировать команды и ответы в файл на ПК.
