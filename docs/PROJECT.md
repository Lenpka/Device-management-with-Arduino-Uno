# Многоканальная система управления исполнительными устройствами (Arduino UNO + Python)

## 1. Название и цель

**Название:** «Разработка программно-аппаратного комплекса дистанционного многоканального управления исполнительными устройствами на базе микроконтроллера Arduino UNO и 8-канального транзисторного драйвера ULN2803A с верхним уровнем на Python».

**Цель:** реализовать двухуровневую систему, в которой Python формирует команды по UART, Arduino UNO выдаёт **логические сигналы управления** на GPIO, ULN2803A выполняет low-side коммутацию цепей **исполнительных устройств**; обеспечить режимы статического включения (`SET`), программного меандра (`FREQ`), аппаратного ШИМ (`PWM` на пинах UNO с hardware PWM) и опрос **логического состояния выходных каналов** (`GET` / `STAT`).

**Задачи:**
- описать архитектуру и обосновать ULN2803A как силовой ключ;
- разработать протокол Python ↔ Arduino;
- реализовать прошивку и клиент;
- испытать на безопасном демонстрационном стенде (см. §6).

---

## 2. Архитектура

```
[ПК: Python]
      │ USB Serial 115200
      ▼
[Arduino UNO]  — 8× логический сигнал управления (TTL)
      ▼
[ULN2803A]     — 8 выходных каналов CH1…CH8, ключ «на землю»
      ▼
[Исполнительные устройства]  ← питание V_load (отдельно от логики 5 V)
```

**Базовый элемент:** выходной канал **CHn** драйвера ULN2803A, а не «реле» и не «абстрактный прибор».  
Реле, сеть 220 В и иные опасные схемы **не входят** в базовую версию проекта. Если на защите нужен электромагнит — только как **исполнительное устройство** на канале, с отдельным питанием, flyback-диодом и обоснованием в приложении; **не** как центральный узел архитектуры.

**Почему схема корректна:** GPIO Arduino не тянет нагрузку по току; ULN2803A — буфер/ключ; V_load и GND разделены с логикой, общая только земля.

## 3. Роль блоков

| Блок | Функция |
|------|---------|
| **Python** | Команды, сценарии, отображение `STAT`, лог на ПК. |
| **Serial** | Канал связи; текстовые команды для отладки. |
| **Arduino UNO** | Парсер; режимы SET / FREQ / PWM; ответы ACK, ERR, STAT. |
| **ULN2803A** | Коммутация тока исполнительного устройства в GND при активном логическом сигнале на INn. |
| **Исполнительное устройство** | LED, малый DC-потребитель и т.д. — между +V_load и OUTn. |

---

## 4. Режимы управления (для защиты)

| Режим | Команда | Суть | Каналы |
|-------|---------|------|--------|
| Статика | `SET` | Постоянный HIGH/LOW | CH1…CH8 |
| Меандр | `FREQ` | Программное переключение, частота в Гц | CH1…CH8 |
| ШИМ | `PWM` | `analogWrite()` — **аппаратный** PWM UNO | CH2, CH4, CH5, CH6, CH7, CH8 |

**Разделение PWM и FREQ:**
- **PWM** — скважность 0…255, частота определяется таймерами ATmega (~490 Hz на типичных пинах UNO). Меняет среднюю мощность на ключе ULN2803 (яркость LED).
- **FREQ** — явная частота меандра 1…100 Гц, реализована в `loop()` через `millis()`. Не вызывает `analogWrite()`.

Подробно: [PROTOCOL.md](PROTOCOL.md), разводка: [WIRING.md](WIRING.md).

## 5. Обратная связь `STAT`

`GET` → `STAT s1,…,s8` — **логическое состояние выходных каналов на пинах Arduino** (HIGH/LOW).

Это **не** телеметрия исполнительного устройства: обрыв LED, КЗ или положение механизма по `STAT` не видны.  
Для обратной связи по нагрузке — датчик тока/напряжения и расширение протокола (опционально).

## 6. Демонстрационный стенд (базовая версия)

**Демонстрируем:**
- 8 выходных каналов CH1…CH8 с **LED + резистор 220–330 Ω**;
- `SET` — включение/выключение;
- `FREQ` — мигание с заданной частотой (например CH1);
- `PWM` — плавное изменение яркости на CH2, CH4, CH5, CH6, CH7, CH8;
- `GET` / `STAT` — опрос логических уровней с ПК.

**Питание:** V_load = **5 V** (БП или стабилизированный блок), **общий GND** с Arduino. USB — питание/прошивка Arduino.

**Не входит в базовую версию:**
- сеть **220 V**;
- более **одной** индуктивной нагрузки без расчёта и flyback-защиты;
- реле как «центральный» элемент схемы.

**Допустимо по согласованию с руководителем:** одна малогабаритная катушка (соленоид 5–12 V) на **одном** канале, отдельный V_load, диод 1N4007 (катод к +V_load), ток в пределах даташита ULN2803A.

## 7. Компоненты

### Обязательные
- Arduino UNO R3
- ULN2803A (модуль)
- USB A–B
- V_load 5 V, общий GND
- 8× LED, 8× резистор, макетка, провода
- ПК, Python 3.10+

### Желательные
- Предохранитель на V_load
- Flyback-диод для **одной** индуктивной демонстрации (если согласовано)
- Логический анализатор (показать разницу FREQ vs PWM)

### Опциональные
- GUI на Python
- Датчик тока на один канал (расширенная обратная связь)
- Внешнее 7–12 V на barrel Arduino при перегрузке USB
