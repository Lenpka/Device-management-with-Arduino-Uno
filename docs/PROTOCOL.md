# Протокол Serial: Python ↔ Arduino UNO

## Параметры линии
- Скорость: **115200** бод  
- Формат: **8 data bits, no parity, 1 stop bit (8N1)**  
- Кодировка команд: **ASCII**  
- Разделитель кадра: **`\n`** (0x0A); символы `\r` игнорируются  

## Команды (хост → устройство)

### `SET <channel> <state>`
- `channel`: целое **1…8**
- `state`: **0** (выкл.) или **1** (вкл.)
- Сбрасывает режим FREQ/PWM на канале, переводит в статический ON/OFF.

Пример: `SET 1 1`

### `FREQ <channel> <hz>`
- `channel`: **1…8**
- `hz`: **0…100** (Гц); **0** — остановить частотное переключение, выход в LOW
- Генерирует меандр ~50 % скважности на выводе канала.

Пример: `FREQ 3 10`

### `PWM <channel> <duty>`
- `channel`: **1…8** (шим только на каналах 2, 4, 5, 8)
- `duty`: **0…255**
- `duty=0` эквивалентно выключению.

Пример: `PWM 5 200`

### `GET`
Запрос снимка состояний всех каналов.

Пример: `GET`

## Ответы (устройство → хост)

### `ACK <эхо команды>`
Успешное выполнение.

Пример: `ACK FREQ 3 10`

### `ERR <код> <детали>`
| Код | Значение |
|-----|----------|
| `SYNTAX` | Неизвестная или malformed команда |
| `CHANNEL` | Номер канала вне 1…8 |
| `PWM` | PWM недоступен на данном канале |
| `FREQ` | Частота вне допустимого диапазона |

Пример: `ERR PWM 1`

### `STAT <s1>,<s2>,...,<s8>`
- `si`: **0** или **1** — текущий логический уровень на выводе Arduino (HIGH = 1)

Пример: `STAT 1,0,0,0,0,0,0,0`

## Порядок работы клиента
1. Открыть порт, подождать **~2 с** (автосброс Arduino при открытии порта).
2. Отправить команду с `\n`.
3. Читать строки до получения `ACK`/`ERR` или `STAT` (таймаут 1 с рекомендуется).
4. Для мониторинга: цикл `GET` каждые 200–500 ms.

## Ограничения реализации (учебный стенд)
- Одновременно на канале активен **один** режим: SET, FREQ или PWM.
- Максимальная частота FREQ зависит от загрузки цикла `loop()`; уточнять осциллографом/логическим анализатором.
