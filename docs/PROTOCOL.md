# Протокол Serial: Python ↔ Arduino UNO

## Параметры линии
- Скорость: **115200** бод  
- Формат: **8 data bits, no parity, 1 stop bit (8N1)**  
- Кодировка команд: **ASCII**  
- Разделитель кадра: **`\n`** (0x0A); символы `\r` игнорируются  

## Термины
- **Выходной канал CHn** — цепь «пин Arduino → INn ULN2803 → OUTn → исполнительное устройство».
- **Логический сигнал управления** — уровень на цифровом выводе Arduino (до драйвера).
- В протоколе номер канала **1…8** соответствует **CH1…CH8**.

## Режимы: SET, FREQ и PWM

На одном канале одновременно активен **один** режим.

### `SET` — статический уровень
- `digitalWrite`: постоянный HIGH (вкл.) или LOW (выкл.).
- Доступен на **всех** выходных каналах CH1…CH8.

### `FREQ` — программный меандр
- Квадратный сигнал с заданной частотой **f** (Гц), скважность ~50 %.
- Реализация: неблокирующее переключение по `millis()` на **любом** цифровом пине.
- **Не** использует `analogWrite()` и **не** является аппаратным ШИМ.
- Доступен на **всех** CH1…CH8.

### `PWM` — аппаратный ШИМ Arduino UNO
- Реализация: `analogWrite(pin, duty)`, duty **0…255**.
- На UNO R3 аппаратный ШИМ только на пинах **3, 5, 6, 9, 10, 11** (документация Arduino).
- В проекте: **CH2, CH4, CH5, CH6, CH7, CH8** (см. [WIRING.md](WIRING.md)).
- Частота ШИМ задаётся таймерами МК (~490 Hz / ~980 Hz в зависимости от пина), **не** параметром команды.
- На CH1 и CH3 команда `PWM` → `ERR PWM`.

## Команды (хост → устройство)

### `SET <channel> <state>`
- `channel`: **1…8** (CH1…CH8)
- `state`: **0** | **1**
- Сбрасывает FREQ/PWM на канале.

Пример: `SET 1 1`

### `FREQ <channel> <hz>`
- `channel`: **1…8**
- `hz`: **0…100**; **0** — останов, выход LOW

Пример: `FREQ 3 10`

### `PWM <channel> <duty>`
- `channel`: **2, 4, 5, 6, 7, 8** (только пины с аппаратным ШИМ)
- `duty`: **0…255**

Пример: `PWM 5 200`

### `GET`
Запрос логического состояния всех выходных каналов.

Пример: `GET`

---

## Ответы (устройство → хост)

### `ACK <эхо>`
Пример: `ACK FREQ 3 10`

### `ERR <код> <детали>`
| Код | Значение |
|-----|----------|
| `SYNTAX` | Неверный формат команды |
| `CHANNEL` | Номер вне 1…8 |
| `PWM` | На канале нет аппаратного ШИМ (CH1, CH3) |
| `FREQ` | Частота вне 0…100 |

Пример: `ERR PWM 1`

### `STAT <s1>,...,<s8>`
- **Логический уровень** на выводе Arduino: **1** = HIGH, **0** = LOW.
- **Не** отражает: исправность LED, ток в цепи, положение механизма.
- При режиме FREQ значения **меняются** между опросами (меандр в реальном времени).
- При PWM на пине — быстрые переключения; `digitalRead` даёт срез уровня в момент опроса.

Пример: `STAT 1,0,0,0,0,0,0,0`

Для мониторинга **исполнительного устройства** нужен отдельный датчик и расширение протокола (вне базовой версии).

---

## Порядок работы клиента
1. Открыть порт, пауза **~2 с** (reset при открытии COM).
2. Отправить команду с `\n`.
3. Читать до `ACK` / `ERR` / `STAT` (таймаут ~1 с).
4. Периодический опрос: `GET` каждые 200–500 ms — только если достаточно знать **логику** на выходах МК.
